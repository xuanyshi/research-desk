About me

I'm Xuanyu Shi, a methods researcher working at the intersection of statistics, data science, and epidemiology — a BS in Statistics and an MS in Data Science (both in the US), and a PhD at Peking University Health Science Center (health big data).

My interests center on causal inference methodology (Pearl's DAG/SCM tradition and the Hernán–Robins epidemiological designs such as target trial emulation), biomedical knowledge graphs, evidence triangulation across study designs (RCT / Mendelian randomization / observational), and real-world evidence — applied mostly in pharmacoepidemiology and cancer epidemiology (e.g. GLP-1RA and cancer, risk factors for gastrointestinal cancers).

Selected work: Nature Communications (2025, first author, Evidence Triangulator), Scientific Data (2024, first author), Health Data Science (2024).

About this site

This site does two things.

It watches a shortlist of strong journals for me. Every day it pulls the new papers, has a model score them against my interests and write a short summary, and files them under Daily. I only read the ones that score well.

And it holds what I've picked out myself. Anything worth keeping goes into the Library with tags, a rating, where I left off, and a note. When I've really worked through a paper, I write it up under Deep reads.

One note: the scores and summaries under Daily are model-generated. The library notes and every deep read are written by me.

Using it

Fetch and score the latest papers:

python -m pipeline.daily

Save a paper:

python -m pipeline.add 2401.12345 --tags causal-inference --rating 4 --status read --note "section 3 is the point"

Rebuild the pages after editing data or templates:

python -m pipeline.render

What it runs on

The pages are static HTML generated with Python and Jinja2, hosted on GitHub Pages. Beyond watching whitelisted journals (Crossref), it also searches OpenAlex by topic across the published literature and tracks arXiv/medRxiv preprints; scoring uses an OpenAI-compatible model endpoint. GitHub Actions handles the daily fetch and each deploy.

The source and configuration notes are in the README.
